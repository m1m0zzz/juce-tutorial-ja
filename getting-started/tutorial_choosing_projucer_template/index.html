チュートリアルProjucer Part 3: アプリケーションに適したProjucerテンプレートの選択

アプリケーションに適切なテンプレートを選択することは、最初のうちは厄介な作業かもしれませんが、最初から適切なクラスを継承することで、生産性が飛躍的に向上します。このチュートリアルでは、Projucerが提供するさまざまなテンプレート・プロジェクトと、アプリケーションを作成する際に注意すべき主なクラスについて説明します。

レベル Beginner

プラットフォーム Windows, macOS, Linux, iOS, Android

クラス： JUCEAアプリケーション, コンポーネント, オーディオコンポーネント, アニメーションAppコンポーネント, OpenGLAppComponent, オーディオプロセッサ, オーディオプロセッサーエディター

Getting started

Projucerをよく理解し、Projucerプロジェクトの構造を基本的に理解していることを確認してください。

If you need help with this step, see チュートリアルProjucerパート1：Projucerを始める.

The right template for the right project

開発するプロジェクトによっては、Projucerウィザードから別のテンプレート・プロジェクトを選びたくなるかもしれません。以下のグラフは、プロジェクトに最適なテンプレートを選択するための視覚的な質問表です：

このブラウザではSVGを表示できません: 代わりにFirefox、Chrome、Safari、またはOperaをお試しください。

すぐに対応するプロジェクトタイプに移動することもできますし、JUCEプロジェクトがどのように構成されているかについての説明を読むこともできます。

The GUI projects

All GUI project templates will provide you with source files to get you started in your development. Since the application will display a graphical user interface, the template provides you with a Main.cpp file in which a class derived from JUCEAアプリケーション will have been created.

The JUCEAアプリケーション class is an abstract base class that provides startup and shutdown functionalities for your applications. The Main.cpp file also creates the application window in which your GUI will reside.

The GUI Application​

The GUI Application project is the most generic of all GUI projects and in addition to the Main.cpp file, it also creates a MainComponent class derived from the コンポーネント class.

The コンポーネント class is a base class for all user-interface objects in JUCE. All your GUI elements should be defined and placed in the MainComponent class.

このブラウザではSVGを表示できません: 代わりにFirefox、Chrome、Safari、またはOperaをお試しください。

GUIアプリケーション・プロジェクトが作成されると、ソース・フォルダーに以下のファイルがあります：

  • Main.cpp: Derived from the JUCEAアプリケーション class, it provides initialisation code for your application and window.
  • MainComponent.h: Header file for the MainComponent derived from the コンポーネント class.
  • MainComponent.cpp: Implementation file for the MainComponent derived from the コンポーネント class.

このプロジェクト・タイプは、アプリケーションにGUIが必要であることはわかっているが、アプリケーションの機能について確信が持てない、あるいは柔軟性が必要な場合に使用します。

注記

For more details on the GUI Application project, please refer to the tutorial チュートリアルアプリケーション・ウィンドウ for more information.

The Audio Application​

The Audio Application is similar to the GUI Application but instead of the MainComponent deriving from the コンポーネント class, it derives from the オーディオコンポーネント class.

The オーディオコンポーネント class is an abstract base class that combines the functionalities of a コンポーネント with an オーディオソース to provide a convenient starting point to handle audio input/output.

このブラウザではSVGを表示できません: 代わりにFirefox、Chrome、Safari、またはOperaをお試しください。

Audio Applicationプロジェクトが作成されると、ソースフォルダに以下のファイルがあります：

  • Main.cpp: Derived from the JUCEAアプリケーション class, it provides initialisation code for your application and window.
  • MainComponent.cpp: Header and implementation file for the MainComponent derived from the オーディオコンポーネント class.

このプロジェクト・タイプは、アプリケーションにGUIによるオーディオ入出力機能が必要な場合に使用します。

注記

For more details on the Audio Application project, please refer to the tutorial チュートリアルホワイトノイズ・ジェネレーターを作る for more information.

The Animated Application​

The Animated Application is similar to the GUI Application but instead of the MainComponent deriving from the コンポーネント class, it derives from the アニメーションAppコンポーネント class.

The アニメーションAppコンポーネント class is an abstract base class that combines the functionalities of a コンポーネント with a タイマー to provide a convenient starting point to display animations.

このブラウザではSVGを表示できません: 代わりにFirefox、Chrome、Safari、またはOperaをお試しください。

アニメーションアプリケーションプロジェクトが作成されると、ソースフォルダに以下のファイルがあります：

  • Main.cpp: Derived from the JUCEAアプリケーション class, it provides initialisation code for your application and window.
  • MainComponent.cpp: Header and implementation file for the MainComponent derived from the アニメーションAppコンポーネント class.

このプロジェクト・タイプは、アプリケーションのGUIに簡単なアニメーションを作成したい場合に使用します。

注記

For more details on the Animated Application project, please refer to the tutorial チュートリアルジオメトリのアニメーション for more information.

The OpenGL Application​

The OpenGL Application is similar to the GUI Application but instead of the MainComponent deriving from the コンポーネント class, it derives from the OpenGLAppComponent class.

The OpenGLAppComponent class is an abstract base class that combines the functionalities of a コンポーネント with an OpenGLRenderer to provide a convenient starting point to render complex graphics.

このブラウザではSVGを表示できません: 代わりにFirefox、Chrome、Safari、またはOperaをお試しください。

OpenGLアプリケーション・プロジェクトが作成されると、ソース・フォルダーに以下のファイルがあります：

  • Main.cpp: Derived from the JUCEAアプリケーション class, it provides initialisation code for your application and window.
  • MainComponent.cpp: Header and implementation file for the MainComponent derived from the OpenGLAppComponent class.

このプロジェクト・タイプは、アプリケーションのGUIでOpenGLを使用して複雑なグラフィック要素をレンダリングする場合に使用します。

注記

For more details on the OpenGL Application project, please refer to the tutorial チュートリアルOpenGLアプリケーションのビルド for more information.

The Command Line project

The Console Application​

コンソールアプリケーションのプロジェクトが作成されると、ソースフォルダに以下のファイルがあります：

  • Main.cpp：Cスタイルのmain()関数を提供し、対応するコマンドライン引数とともにアプリケーションを実行します。

コマンドラインでバイナリが実行されると、以下の関数が呼び出される：

int main (int argc, char* argv[])
{
 
    // ...あなたのコードはここにある！
 
 
    0 を返します；
}

このプロジェクト・タイプでは、Main.cpp ファイルにクラスは作成されず、ベース・クラスも継承されませんが、.jucer ファイルと JUCE ライブラリ・コード・フォルダが作成されます。

このプロジェクト・タイプは、アプリケーションにGUIを必要とせず、コマンドライン・コンソールでアプリケーションを実行したい場合に使用します。

The Plugin project

The Audio Plugin​

The Audio Plugin project has an all around different project structure compared to other template projects offered by the Projucer and creates an オーディオプロセッサ and an オーディオプロセッサーエディター.

The オーディオプロセッサ class is an abstract base class that handles the audio processing of your audio plugin. It represents an instance of a loaded plugin and is wrapped by the different plugin formats such as VST, AU, RTAS and AAX.

The オーディオプロセッサーエディター class is a base class derived from the コンポーネント class and holds the GUI functionalities of your audio plugin.

このブラウザではSVGを表示できません: 代わりにFirefox、Chrome、Safari、またはOperaをお試しください。

Audio Pluginプロジェクトが作成されると、ソースフォルダに以下のファイルがあります：

  • PluginProcessor.h: Header file for the PluginProcessor derived from the オーディオプロセッサ class.
  • PluginProcessor.cpp: Implementation file for the PluginProcessor derived from the オーディオプロセッサ class.
  • PluginEditor.h: Header file for the PluginEditor derived from the オーディオプロセッサーエディター class.
  • PluginEditor.cpp: Implementation file for the PluginEditor derived from the オーディオプロセッサーエディター class.

このプロジェクトタイプは、DAWでホストできるプラグインや、GUIを備えたスタンドアロンアプリケーションを作成する場合に使用します。

注記

For more details on the Audio Plugin project, please refer to the tutorial チュートリアル基本的なAudio/MIDIプラグインを作る, パート1：セットアップ for more information.

The Library projects

The Static/Dynamic Library​

他のプロジェクトで使用するライブラリを作成することに興味がある場合、Projucerは静的および動的ライブラリを作成するオプションを提供します。

このプロジェクトタイプでは、ソースファイルは作成されませんが、.jucerファイルとJUCEライブラリコードフォルダが作成されます。

JUCEのモジュールにいつでもアクセスできる利便性で、ファイルを自由に追加することができます。

  • JUCEの様々なクラスは、そのような様々なプロジェクトを促進する。
  • JUCEフレームワークの基本的な仕組みを理解。
