チュートリアルデシベルを使ってオーディオレベルをコントロールする

このチュートリアルでは、デシベルスケールを使ってオーディオの出力レベルを変更する方法を示します。これは、オーディオアプリケーションでオーディオレベルの値をユーザーに表示する、より一般的な方法です。

レベル：中級

プラットフォーム：Windows, macOS, Linux

クラス： デシベル, スライダー, ストリング

はじめる​

Download the demo project for this tutorial here: ピップ | ジップ. Unzip the project and open the first header file in the Projucer.

If you need help with this step, see チュートリアルProjucerパート1：Projucerを始める.

注記

This tutorial leads on from チュートリアルオーディオレベルのコントロール. You should have followed that tutorial before continuing.

The demo project

In a similar way to the チュートリアルオーディオレベルのコントロール demo project, the demo project for this tutorial presents a window containing a single slider. This time the slider value is represented in デシベル. This value in デシベル needs to be converted to a linear gain value before being used in the audio processing algorithm. Most audio applications express gain to users in デシベル as this often フィーリング more natural as the values are varied (or compared). The user interface for the demo project is shown in the following screenshot.

Creating a customised slider

Notice this time that the text displayed adjacent to the slider not only shows the value in デシベル but it also shows the suffix "dB". This is achieved by creating a custom slider class, デシベルスライダー that inherits from the スライダー class. In this custom slider class the text box interface is customised to display the value in デシベル. While a suffix to the text displayed within a スライダー object's text box can be added using the スライダー::setTextValueSuffix() function, we need one more customisation. This is to customise the way that values are converted such that we can display -INF dB when the level drops very low.

注記

INF refers to インフィニティ where -INF dB is treated as silence for the purposes of our application.

The Decibels class​

The デシベル class contains a number of static functions necessary for the conversion between the values in デシベル and linear gain. It also provides a simple means of converting values in デシベル to a ストリング object. For example, we override the virtual function スライダー::getTextFromValue() by using the デシベル::toString() function (in the デシベルスライダー class) like so:

juce::String getTextFromValue (double value) オーバーライド
    {
        juce::Decibels::toString (value) を返します；
    }

This allows our デシベルスライダー class to display the appropriate text in its text box for a given slider value.

The デシベル class doesn't have a function to convert a ストリング object back to a value in デシベル, so we need to write our own. Here we override the スライダー::getValueFromText() virtual function (again in the デシベルスライダー class):

double getValueFromText (const juce::String& text) override
    {
        auto minusInfinitydB = -100.0；
 
        auto decibelText = text.upToFirstOccurrenceOf ("dB", false, false).trim(); // [1].
 
        return decibelText.equalsIgnoreCase ("-INF") ? minusInfinitydB
                                                     : decibelText.getDoubleValue(); // [2].
    }

これは、ユーザーがテキストボックスに値を入力し、それがチェックされ、スライダーの有効な値に変換されることを可能にします。これを行うには

  • [1]: Strip off the "dB" suffix from the text (if it is present), and trim the text to remove any leading or trailing whitespace (using the 文字列::トリム() function).
  • [2]: Check if the remaining text is "-INF" and return -100 in this case (this is the default value in デシベル for -INF dB that the デシベル class uses, see the 備考 for this tutorial for a discussion on this). Otherwise we convert this remaining text to a ダブル value and return it.

Using the slider value

In the チュートリアルオーディオレベルのコントロール we accessed the slider's value directly in the getNextAudioBlock() function. Since the conversion from デシベル to linear gain involves some potentially CPU-intensive arithmetic, it would be wise to avoid performing the conversion too often, especially on the audio thread. In the demo project for this tutorial we store a フロート member レベル in the メインコンテンツコンポーネント class and update this value when the slider changes using a listener.

We initialise the レベル member to zero in the constructor and convert this to デシベル using the デシベル::gainToDecibels() function to give the slider its initial position (using the スライダー::setValue() function) like so:

MainContentComponent()
    {
decibelSlider.setValue (juce::Decibels::gainToDecibels (level))；
        decibelLabel.setText ("Noise Level in dB", juce::dontSendNotification)；

Processing the audio​

In our MainContentComponent::getNextAudioBlock() we process the audio:

void getNextAudioBlock (const juce::AudioSourceChannelInfo& bufferToFill) override
    {
        auto currentLevel = level；
        auto levelScale = currentLevel * 2.0f；
 
        for (auto channel = 0; channel < bufferToFill.buffer->getNumChannels(); ++channel)
        {
            auto* buffer = bufferToFill.buffer->getWritePointer (channel, bufferToFill.startSample)；
 
            for (auto sample = 0; sample < bufferToFill.numSamples; ++sample)
                buffer[sample] = random.nextFloat() * levelScale - currentLevel；
        }
    }

Note that this is almost identical to the getNextAudioBlock() function from チュートリアルオーディオレベルのコントロール ただし that we just take a function-local copy of the レベル value then calculate our レベルスケール value as before.

Issues with this approach​

One issue with this approach is that the レベル variable might change its value abruptly during the execution of the audio thread (in this case, in between two calls to ゲットネクストオーディオブロック). Such changes will typically introduce audio artefacts such as an audible crackling. To avoid this, in a real-world synthesiser we would want a レベル value that changes smoothly. Techniques to accomplish this are explored in other tutorials (see チュートリアルサイン波シンセサイザーを作る).

Another issue is related to thread safety. Writing to a member variable like レベル in one thread (the GUI thread) and reading the same value from another thread (the audio thread) is technically undefined behaviour in C++ if no thread synchronisation takes place (either via critical sections or using atomics). These issues are beyond the scope of this tutorial and will be discussed in a future tutorial. In this particular case we don't have to worry about this too much, because on typical architectures (x86, x86_64, ARM) reading and writing a single フロート is an atomic operation: the reads and writes cannot be intermingled and are generally safe.

Thinking further, it might be tempting to optimise the code further by making レベルスケール a member variable too (and therefore not calculate it for every call to the getNextAudioBlock() function). But then the two member variables would not be updated as a single atomic operation anymore. There are, of course, ways around this as well, but this is beyond the scope of this tutorial.

Notes

The code presented in the demo project for this tutorial assumes that we want to treat a value of -100 dB or lower as -INF dB (that is a linear gain value of zero). This value of -100 dB is the default value used by the デシベル class but you can override this in its calculations. This is achieved by providing an additional argument to each of the functions in the デシベル class specifying which value should be treated as -INF dB. For example, to use -96 dB (and below) as -INF dB when updating our slider value in the メインコンテンツコンポーネント constructor we could do this:

decibelSlider.setValue(juce::Decibels::gainToDecibels(レベル, -96.0f))；

But of course we need to ensure that all parts of our application use the same value for -INF dB. There is one potential problem with the code for the demo project since we hard-code -100.0 in our デシベルスライダー::getValueFromText() function. If the デシベル class changes its default value (for some reason) then our code would break. Unfortunately, this default value is a private member of the デシベル class, so we can't ask the デシベル class for its default value. Instead, we would need to specify our own default value and use this throughout.

Modify the demo project to specify your own default value for -INF as -96 dB. You should need to update both the デシベルスライダー class and the メインコンテンツコンポーネント class.
