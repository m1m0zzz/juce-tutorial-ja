チュートリアルAudioDeviceManagerクラス

This tutorial introduces the オーディオデバイスマネージャー class which is used for managing audio devices on all platforms. This allows you to configure things such as the device sample rate and the number of inputs and outputs.

レベル：中級

プラットフォーム：Windows, macOS, Linux, iOS, Android

クラス： オーディオデバイスマネージャー, オーディオデバイスセレクタコンポーネント, 変更リスナー, BigInteger

はじめる​

注記

This tutorial assumes that you have a basic grasp of using the オーディオコンポーネント class for audio input and output (see チュートリアルオーディオ入力を処理する). It also assumes that you are familiar with basic GUI layout techniques (see チュートリアル親子コンポーネント).

Download the demo project for this tutorial here: ピップ | ジップ. Unzip the project and open the first header file in the Projucer.

If you need help with this step, see チュートリアルProjucerパート1：Projucerを始める.

The demo project

The demo project is based on the Audio Application template from the Projucer. It presents an オーディオデバイスセレクタコンポーネント object which allows you to configure your audio device settings. The demo project also presents a simple text console that reports the current audio device settings. The application also shows the current CPU usage of the audio processing element of the application.

注記

The code presented here is broadly similar to the オーディオ設定デモ from the JUCE Demo. The main difference is that the audio generated is the same as in the チュートリアルオーディオ入力を処理する (this ring-modulates the audio input with white noise).

Audio Devices

JUCEは、それがサポートする全てのプラットフォーム上でオーディオデバイスにアクセスする一貫した手段を提供します。ここで提供されるデモアプリケーションは、デスクトッププラットフォームにのみデプロイされるかもしれませんが、これはGUIレイアウトの制約によるものです。オーディオはモバイルプラットフォームでもシームレスに動作します。

In the Audio Application template, the オーディオコンポーネント class instantiates an オーディオデバイスマネージャー object デバイスマネージャー — it is a public member, therefore it is accessible from your subclass. The オーディオコンポーネント class also performs some basic initialisation of this オーディオデバイスマネージャー object — this happens when you call AudioAppComponent::setAudioChannels().

The オーディオデバイスマネージャー class will attempt to use the default audio device unless this is overridden. This may be configured in code or via the オーディオデバイスセレクタコンポーネント as illustrated here. Device settings and preferences may be stored and recalled on subsequent applications launches. The オーディオデバイスマネージャー class can also fallback to the default device in such circumstances if the preferred device is no longer available (for example, if an external audio device has been unplugged since the last launch).

The オーディオデバイスマネージャー class is also a hub for incoming MIDI messages. This is explored in other tutorials (see チュートリアル：MIDIイベントの処理).

The オーディオデバイスマネージャー class can broadcast changes to its settings as it inherits from the チェンジブロードキャスター class. The right-hand side of our component posts some of the important audio device settings whenever a change in our オーディオデバイスマネージャー object is triggered.

The AudioDeviceSelectorComponent class​

The オーディオデバイスセレクタコンポーネント class provides convenient way of configuring audio devices on all platforms. As mentioned above, this is displayed in the right-hand side of our user interface for the demo project. When the オーディオデバイスセレクタコンポーネント object is constructed we pass it the オーディオデバイスマネージャー object that we want it to control along with a number of other options including the number of channels to support (see the オーディオデバイスセレクタコンポーネント class ビルダー for more information). Here we create the オーディオデバイスセレクタコンポーネント object by passing it the オーディオデバイスマネージャー object that is the member of the オーディオコンポーネント, allowing up to 256 input and output channels, hiding MIDI configuration, and showing our channels as single channels rather than stereo pairs:

  • オーディオ機器の変化に対応。
  • サンプルレートやチャンネル数など、オーディオデバイスの便利な設定にアクセスできます。
